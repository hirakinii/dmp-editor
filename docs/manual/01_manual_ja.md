## 1. はじめに

本マニュアルは、公的研究資金の提供機関から要求されるデータマネジメントプラン（DMP）の作成支援ツール「DMP Editor」の操作手順をまとめたものです。

本システムは GakuNin RDM（GRDM）と連携して動作するため、利用には GRDM のアカウントおよびアクセストークンが必要です。

**DMP Editor の主な機能:**

| 機能 | 説明 |
|------|------|
| DMP の作成・編集 | 5 つのステップ形式で DMP を入力・管理する |
| KAKEN 自動補完 | 科研費課題番号を入力するとプロジェクト情報・研究者情報を自動取得する |
| ROR 機関検索 | 機関名でデータ管理機関を検索し、ROR ID を自動設定する |
| GRDM ユーザー検索 | GRDM 登録ユーザーを姓で検索して担当者情報を自動取り込む |
| GRDM ファイル連携 | GRDM 上の実データファイルを DMP データ項目に紐付ける |
| Excel エクスポート | 汎用 Excel 形式・JSPS 様式 Excel 形式でエクスポートする |

## 2. 事前準備：GRDM Token の取得

ログインに必要なトークンをお持ちでない場合は、以下の手順で発行してください。

1. **設定画面へのアクセス**
   Web ブラウザで GakuNin RDM のトークン設定ページ（`https://rdm.nii.ac.jp/settings/tokens`）にアクセスします。
2. **新規トークンの作成**
   「パーソナルアクセストークン」の画面にて、「新規トークン」を選択します。
3. **トークン情報の入力**
   - **トークン名**: 任意の名前を入力します（例: `dmp-editor`）。
   - **スコープ（権限設定）**: **必ず `osf.full_write` にチェックを入れてください**。
     - ※ `osf.full_write` の権限が付与されていない場合、正しく連携できません。
4. **作成の完了**
   「作成」ボタンをクリックしてトークンを生成し、表示されたトークン文字列をコピーして控えてください。

## 3. ログイン（認証）

1. **DMP Editor トップページへアクセス**
   DMP Editor を開き、「GRDM との接続」画面を表示します。
2. **トークンの入力**
   「GRDM Token」入力欄に、取得したトークンを貼り付けます。
3. **認証の実行**
   「認証」ボタンをクリックします。入力されたトークンはブラウザのローカルストレージにのみ保存され、GRDM との通信にのみ利用されます。サーバーへの送信は行いません。

認証に成功すると、ヘッダーにログインユーザーの氏名・所属機関が表示され、プロジェクト一覧画面へ遷移します。

## 4. プロジェクト一覧画面の操作

ログイン後、「DMP Project 一覧」が表示されます。

| 操作 | 説明 |
|------|------|
| **一覧の確認** | 作成済みの DMP プロジェクト名・作成日・最終更新日を確認できます |
| **新規作成** | 「新規 DMP Project を作成する」ボタンをクリックして新しい DMP を作成します |
| **編集** | 各行の「編集」ボタンをクリックして DMP 編集画面を開きます |
| **詳細表示** | 各行の「詳細」ボタンをクリックして DMP の内容を読み取り専用で閲覧します |
| **GRDM リンク** | 各行のリンクから GRDM 本体の該当プロジェクトページへ直接アクセスできます |

## 5. DMP の新規作成・編集

「DMP Project 編集」画面は **5 つのステップ**で構成されています。画面上部のステップバーをクリックするか、「次へ」「戻る」ボタンで各ステップを移動します。各ステップの必須項目が未入力の場合、次のステップへ進めません。

入力内容はステップをまたいで保持されます。編集完了後は最下部の「保存」ボタンで GRDM に保存してください。

> **表の見方**: ✅ = 必須、— = 任意

#### KAKEN 番号による自動補完

科研費課題番号（例: `23K12345`）を入力フィールドに入力して検索すると、KAKEN API から以下の情報を自動取得してフォームに反映します：

- 資金配分機関名・プログラム名・プログラムコード
- 採択年度・事業開始年度・事業終了年度
- 研究代表者・研究分担者の情報（ステップ 3 に反映）

取得結果は **確認ダイアログ（KakenConfirmDialog）** で表示されます。ダイアログにはプログラム名・プロジェクト名・採択年度・取得された研究者一覧が表示されるので、内容を確認してから「適用」をクリックしてください。

担当者が既にステップ 3 に登録されている場合、同姓同名・同所属の担当者が重複しないよう **重複確認ダイアログ（DuplicatePersonDialog）** が表示されます。重複する担当者を個別にスキップすることができます。

DMP に関与する担当者を複数登録できます。「担当者を追加」ボタンをクリックして入力フォームを展開し、情報を入力します。登録済みの担当者はドラッグ操作で並べ替えが可能です。

| 姓 | 担当者の姓 | ✅ | ✅ | ✅ |
| 名 | 担当者の名 | ✅ | ✅ | ✅ |
| 所属機関 | 担当者の所属機関名 | ✅ | ✅ | ✅ |
| e-Rad 研究者番号 | 府省共通研究開発管理システムの研究者 ID | — | — | — |
| ORCID | 国際的な研究者識別子 | — | — | — |
| 連絡先（メールアドレス） | 担当者のメールアドレス | — | — | — |
| GRDM ユーザー ID | GRDM アカウントとの紐付け用識別子（GRDM 検索時に自動設定） | — | — | — |

> **制約**: 「研究代表者」および「管理対象データの管理責任者」はそれぞれ 1 名のみ登録できます。また、同姓同名・同所属の担当者の重複登録はできません。

#### GRDM ユーザー検索

「GRDM ユーザーを検索」パネルを展開すると、GRDM に登録されているユーザーを**姓**で検索できます。候補から選択すると、氏名・所属機関・ORCID・e-Rad 番号・GRDM ユーザー ID が自動的に入力されます。

#### 値の出典表示

各担当者レコードには、情報の取得元を示すチップが表示されます：
- **KAKEN**: KAKEN API から自動取得した値
- **GRDM**: GRDM ユーザー検索から取得した値
- （チップなし）: ユーザーが手動入力した値

#### ROR 機関検索

「データ管理機関」フィールドの隣にある検索アイコンをクリックすると、ROR（Research Organization Registry）API を使って機関名を検索できます。検索結果から機関を選択すると、機関名と ROR ID が自動的に入力されます。

- 日本語キーワードでの検索に対応しています。
- 2 文字以上入力すると候補が表示されます（300ms デバウンス）。

#### GRDM ファイルのリンク

各データセットの編集フォーム内にある「**GRDM ファイルを選択**」ボタンをクリックすると、ステップ 5 で関連付けられた GRDM プロジェクトのファイルツリーが表示されます。ここからリンクするファイルを選択することで、データセット定義と実際のファイルを紐付けることができます。

- フォルダを選択すると、そのフォルダ内の全ファイルが一括でリンクされます。
- 複数ファイルを選択できます。

リンク後、「**GRDM メタデータを比較**」ボタンをクリックすると、リンクしたファイルの GRDM 上のメタデータ（名称・分野・種別・アクセス権等）と現在の入力値を並べて比較できます。GRDM 側の値を選択して取り込むことが可能です。取り込んだフィールドには「GRDM」の出典チップが表示されます。

### ステップ 5: GRDM 連携

DMP プロジェクトに対し、実データが格納されている GRDM プロジェクトを関連付け、ファイルを DMP データ項目にマッピングします。

#### GRDM プロジェクトの関連付け

プルダウンメニューから関連付ける GRDM プロジェクトを選択します。複数のプロジェクトを関連付けることができます。

#### ファイルツリーの参照とマッピング

関連付けた GRDM プロジェクトのファイル・フォルダ構造がツリー形式で表示されます。各データセット（ステップ 4 で登録）に対して、対応する GRDM ファイルを紐付けることができます。

紐付けられたファイルについては、以下のメタデータが DMP データ内に自動記録されます：
- ファイル名・フルパス・サイズ
- MD5 / SHA256 ハッシュ値
- 作成日・更新日・最終アクセス日時
- GRDM ページ URL

> **ヒント**: ステップ 4（研究データ情報）の各データセット編集フォームからも、ファイルの選択・GRDM メタデータとの比較が行えます。

---

### 保存

編集内容は画面最下部の「**保存**」ボタンをクリックして GRDM に保存します。保存成功時はトースト通知が表示されます。

> **注意**: フォームに未保存の変更がある状態でページを離脱しようとすると、確認ダイアログが表示されます。意図しないデータ消失を防ぐため、必ず「保存」を実行してからページを離脱してください。

## 6. DMP の出力（エクスポート）

作成した DMP は Excel ファイルとして出力できます。プロジェクト一覧画面の各行にある「**エクスポート**」ボタン、または DMP 詳細表示画面の「**エクスポート**」ボタンをクリックしてください。

### 汎用 Excel 形式（.xlsx）

DMP の全フィールドを以下の 4 シートに展開してエクスポートします。機関横断的な記録・共有・バックアップ用途に適しています。

| シート | 内容 |
|--------|------|
| DMP 情報 | 種別・提出日・作成日・更新日 |
| プロジェクト情報 | 資金配分機関・プログラム名・体系的番号・研究期間等 |
| 担当者情報 | 役割・氏名・所属・識別子等 |
| 研究データ情報 | 全データセットの詳細フィールド |

### JSPS 様式 Excel 形式（.xlsx）

日本学術振興会（JSPS）の DMP 提出様式テンプレートに DMP データを埋め込み、そのまま提出可能な Excel ファイルとして出力します。

- 担当者・データ項目数が様式のデフォルト行数を超える場合は、自動的に行が追加されます。
- セルの書式（色・フォント等）は保持されます。
